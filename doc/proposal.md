# 校园论坛网站 — 需求文档

> 版本：v1.0 | 日期：2026-05-27

---

## 一、项目概述与目标

### 1.1 项目背景

为在校学生提供一个专属的校园交流平台，涵盖日常交流、二手交易、失物招领、表白墙、学习讨论等场景，替代分散在微信群/QQ群中的碎片化信息，形成可沉淀、可检索的校园社区。

### 1.2 项目目标

| 目标 | 说明 |
|------|------|
| 信息聚合 | 将校园各类信息集中到一个平台，便于浏览和检索 |
| 身份可信 | 仅限本校在校学生注册，确保社区成员真实可靠 |
| 移动友好 | 支持手机浏览器访问，适配移动端使用场景 |
| 易于维护 | 个人可独立开发和运维，架构简洁清晰 |

### 1.3 预期规模

- 用户量级：数千人（单校）
- 并发峰值：数百人同时在线

---

## 二、用户画像与使用场景

### 2.1 用户画像

| 角色 | 描述 |
|------|------|
| 普通学生 | 注册用户，可浏览、发帖、评论、点赞、私信 |
| 管理员 | 平台维护者（项目作者），负责内容审核、板块管理、用户管理 |

### 2.2 典型使用场景

1. **信息发布**：学生发布二手物品、失物招领、活动通知等帖子
2. **讨论交流**：在学习交流板块讨论课程、考试、技术问题
3. **表白墙**：匿名或实名发布表白信息
4. **信息检索**：通过搜索功能查找历史帖子和有用信息
5. **私信沟通**：对感兴趣的帖子联系发布者进行私下沟通

## 三、功能需求

### 3.1 核心功能

#### 3.1.1 用户系统

| 功能 | 说明 |
|------|------|
| 注册 | 使用学号 + 邮箱注册，需邮箱验证码验证身份 |
| 登录/登出 | 学号 + 密码登录，支持 Token 认证 |
| 个人信息 | 修改昵称、头像、个人简介 |
| 密码管理 | 修改密码、忘记密码（邮箱找回） |

> **身份验证策略**：通过学校邮箱后缀（`@xxx.edu.cn`）限制注册范围，确保仅本校学生可注册。

#### 3.1.2 板块/分区

| 功能 | 说明 |
|------|------|
| 预设板块 | 校园公告、二手交易、失物招领、表白墙、学习交流、灌水区 |
| 板块浏览 | 按板块分类浏览帖子列表 |
| 板块管理 | 管理员可新增、编辑、删除、排序板块 |

#### 3.1.3 帖子系统

| 功能 | 说明 |
|------|------|
| 发帖 | 选择板块，填写标题、正文（富文本），支持上传图片 |
| 编辑/删除 | 作者可编辑、删除自己的帖子 |
| 帖子列表 | 按时间/热度排序，支持分页 |
| 帖子详情 | 查看完整内容、评论列表 |
| 置顶/精华 | 管理员可将帖子设为置顶或精华 |

#### 3.1.4 评论系统

| 功能 | 说明 |
|------|------|
| 发表评论 | 对帖子发表评论 |
| 回复评论 | 支持楼中楼（一级回复） |
| 删除评论 | 作者可删除自己的评论 |

#### 3.1.5 互动功能

| 功能 | 说明 |
|------|------|
| 点赞 | 对帖子和评论点赞/取消点赞 |
| 收藏 | 收藏帖子，可在个人中心查看 |

#### 3.1.6 私信系统

| 功能 | 说明 |
|------|------|
| 发送私信 | 向其他用户发送私信 |
| 私信列表 | 查看会话列表和聊天记录 |
| 未读提醒 | 导航栏显示未读私信数量 |

#### 3.1.7 搜索功能

| 功能 | 说明 |
|------|------|
| 关键词搜索 | 按标题/内容搜索帖子 |
| 板块内搜索 | 在指定板块内搜索 |
| 按用户搜索 | 查看某用户发布的所有帖子 |

#### 3.1.8 管理后台

| 功能 | 说明 |
|------|------|
| 用户管理 | 查看用户列表、禁用/启用用户 |
| 内容审核 | 审核帖子、删除违规内容 |
| 板块管理 | 增删改板块 |
| 数据统计 | 用户数、帖子数、活跃度等基础统计 |

---

### 3.2 扩展功能（后续迭代）

| 功能 | 说明 | 优先级 |
|------|------|--------|

## 四、非功能需求

### 4.1 性能

| 指标 | 要求 |
|------|------|
| 首页加载 | < 2 秒（本地网络环境） |
| 接口响应 | 核心接口 < 500ms |
| 并发支持 | 支持数百人同时在线 |
| 数据库查询 | 关键查询需建立索引，避免全表扫描 |

### 4.2 安全

| 项目 | 要求 |
|------|------|
| 身份认证 | JWT Token 认证，Token 有过期机制 |
| 密码存储 | 使用 BCrypt 等算法加密存储 |
| 输入校验 | 前后端双重校验，防止 XSS、SQL 注入 |
| 权限控制 | 普通用户仅能操作自己的数据，管理员有独立权限 |
| 接口防护 | 限流防刷，敏感操作需二次验证 |
| CORS | 配置跨域策略，仅允许前端域名访问 |

### 4.3 兼容性

| 项目 | 要求 |
|------|------|
| 浏览器 | Chrome、Firefox、Safari、Edge（近两个主要版本） |
| 移动端 | 适配 iOS Safari、Android Chrome |
| 响应式 | 断点设计：手机（< 768px）、平板（768-1024px）、桌面（> 1024px） |

### 4.4 可用性

| 项目 | 要求 |
|------|------|
| 界面风格 | 简洁清新，符合年轻学生审美 |
| 操作反馈 | 加载状态、操作成功/失败提示 |
| 错误处理 | 友好的错误页面和提示信息 |

## 五、约束条件

### 5.1 技术栈

| 层次 | 技术选型 |
|------|----------|
| 前端框架 | Vue 3 + Vite |
| UI 组件库 | Element Plus 或类似组件库 |
| 状态管理 | Pinia |
| 路由 | Vue Router |
| HTTP 客户端 | Axios |
| 后端框架 | Spring Boot 3 |
| ORM | MyBatis-Plus 或 JPA |
| 数据库 | PostgreSQL |
| 认证 | Spring Security + JWT |
| 接口文档 | Swagger / Knife4j |
| 文件存储 | 本地文件系统（初期） |

### 5.2 开发与部署

| 项目 | 说明 |
|------|------|
| 开发环境 | 本地开发，前后端分离 |
| 版本管理 | Git |
| 包管理 | 前端 npm/pnpm，后端 Maven/Gradle |
| 部署方式 | 初期本地运行，后续可部署到服务器 |

### 5.3 时间规划（建议）

| 阶段 | 内容 | 周期 |
|------|------|------|
| 第一阶段 | 用户系统 + 帖子系统 + 板块 + 评论 | 3-4 周 |
| 第二阶段 | 点赞/收藏 + 搜索 + 私信 | 2-3 周 |
| 第三阶段 | 管理后台 + 内容审核 | 2 周 |
| 第四阶段 | 扩展功能 + 优化 + 测试 | 2-3 周 |

## 六、数据库设计（概要）

### 核心表

| 表名 | 说明 |
|------|------|
| `user` | 用户表（学号、昵称、密码、头像、邮箱、角色、状态） |
| `board` | 板块表（名称、描述、排序、状态） |
| `post` | 帖子表（标题、内容、作者、板块、图片、置顶、精华、状态） |
| `comment` | 评论表（内容、作者、帖子、父评论、状态） |
| `like` | 点赞表（用户、目标类型、目标ID） |
| `favorite` | 收藏表（用户、帖子） |
| `message` | 私信表（发送者、接收者、内容、已读状态） |
| `notification` | 通知表（用户、类型、内容、已读状态） |

### ER 关系（简述）

```
User 1---N Post
User 1---N Comment
Post 1---N Comment
Comment 1---N Comment (回复)
User N---N Post (点赞/收藏)
User 1---N Message (发送/接收)
Board 1---N Post
```

## 七、API 设计（概要）

### 接口风格：RESTful

| 模块 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 认证 | POST | `/api/auth/register` | 注册 |
| 认证 | POST | `/api/auth/login` | 登录 |
| 认证 | POST | `/api/auth/logout` | 登出 |
| 用户 | GET | `/api/users/me` | 获取当前用户信息 |
| 用户 | PUT | `/api/users/me` | 更新个人信息 |
| 板块 | GET | `/api/boards` | 获取板块列表 |
| 帖子 | GET | `/api/posts` | 获取帖子列表（支持筛选、分页） |
| 帖子 | POST | `/api/posts` | 发布帖子 |
| 帖子 | GET | `/api/posts/{id}` | 获取帖子详情 |
| 帖子 | PUT | `/api/posts/{id}` | 编辑帖子 |
| 帖子 | DELETE | `/api/posts/{id}` | 删除帖子 |
| 评论 | GET | `/api/posts/{id}/comments` | 获取评论列表 |
| 评论 | POST | `/api/posts/{id}/comments` | 发表评论 |
| 点赞 | POST | `/api/likes` | 点赞/取消点赞 |
| 收藏 | POST | `/api/favorites` | 收藏/取消收藏 |
| 私信 | GET | `/api/messages` | 获取会话列表 |
| 私信 | POST | `/api/messages` | 发送私信 |
| 搜索 | GET | `/api/search` | 搜索帖子 |
| 管理 | GET | `/api/admin/users` | 用户管理 |
| 管理 | PUT | `/api/admin/posts/{id}/pin` | 置顶帖子 |
| 管理 | PUT | `/api/admin/posts/{id}/feature` | 设为精华 |

## 八、页面结构（概要）

| 页面 | 路由 | 说明 |
|------|------|------|
| 首页 | `/` | 热门帖子、公告、板块入口 |
| 板块页 | `/board/:id` | 某板块的帖子列表 |
| 帖子详情 | `/post/:id` | 帖子内容 + 评论 |
| 发帖页 | `/post/create` | 发布新帖子 |
| 登录页 | `/login` | 登录表单 |
| 注册页 | `/register` | 注册表单 |
| 个人中心 | `/profile` | 个人信息、我的帖子、收藏 |
| 私信页 | `/messages` | 会话列表 + 聊天窗口 |
| 搜索结果 | `/search` | 搜索结果列表 |
| 管理后台 | `/admin/*` | 用户管理、内容管理、数据统计 |
